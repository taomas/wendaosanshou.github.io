git介绍

git是一种分布式版本控制系统（Distributed Version Control System，简称 DVCS），而SVN是非分布式版本控制系统。

分布式版本控制系统

拥有一个服务端的代码仓库，客户端每次clone都是将远程仓库的代码完整的镜像出来。这样一来，如果服务端的仓库发生故障，事后可以将任何一处镜像出来的本地仓库恢复。
在没有网络的情况下，客户端仍然能够正常的修改，提交代码。

集中式版本控制系统

也拥有一个单一的集中管理的服务器，保存所有文件的修订版本。协同工作的人需要通过客户端连接到服务器，取出最新的代码或者提交更新。
这样做的好处是可以一定程度的看到其他人在做什么。坏处是如果服务器宕机一个小时，这一个小时内任何人都不能够提交代码。

用户信息

配置

用户安装完成git后，第一件事就是配置你的用户名和邮箱地址。因为每次git提交都会使用这些信息。

如果想要检查你的配置，可以使用 git config --list 命令来列出所有 Git 当时能找到的配置。

也可以通过git config <key>来查看某一项的配置

git常用命令行操作

1.修改并提交代码

查看代码状态：git status
查看状态简写方式：git status -s
添加所有已修改代码到暂存区：git add .
提交代码并且添加提交信息：git commit -m "first commit"
推送代码到远程仓库：git push origin master

2.忽略文件

创建一个名为.gitignore的文件，列出要忽略的文件模式。
查看一个 .gitignore 文件的例子：

3.撤销操作

重新提交：git commit --amend
比如第一次提交忘记添加某个文件，需要再次提交：

取消暂存中的某一个文件：git reset HEAD <filename>

撤销对文件的任何修改：git checkout -- <filename>

4.远程仓库操作

查看远程仓库：git remote
你也可以指定选项 -v，会显示需要读写远程仓库使用的 Git 保存的简写与其对应的 URL。

添加远程仓库：git remote add origin <url>

5.打标签

列出已有标签:git tag

查看标签信息：git show <tag>

创建附注标签：git tag -a <v1.0> -m "my version 1.0"

创建轻量标签：git tag <v1.0>

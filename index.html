内部方法

api方法

each

_.each(list, iteratee, [context])
遍历list中每一个元素，按顺序遍历输出每一个元素，在iteratee中传递三个参数(element , index , list)，如果list为对象，则传递三个参数(key , value , list)。
如果传递了context参数，则将iteratee方法绑定到context对象上。
each方法只遍历对象或数组，不返回执行后结果，返回原数组或原对象。

map

_.map(list, iteratee, [context])
遍历list中元素，执行iteratee

内部方法cb

这里有个内部方法cb(),用来优化回调方法iteratee

find

_.find(list, predicate, [context])
在list中逐项查找，返回第一个通过predicate迭代函数真值检测的值，如果没有任何通过predicate迭代函数的值，返回undefined.
如果找到通过predicate迭代函数的值，函数立即返回该值，不会遍历整个list。

这里有用到findIndex查找数组中匹配的值的index，用findKey查找对象中匹配的值的key

findIndex

_.findIndex(array, predicate, [context])，类似于_.indexOf(),当predicate通过真检测时，返回第一个index索引，否则返回-1

findKey

_.findKey(array, predicate, [context])，当predicate通过真检测时，返回key

filter

_.filter(list, predicate, [context])
遍历list中的每个值，返回所有通过predicate真值检测的值组成的数组

reject

_.reject(list, predicate, [context])
返回list中没有通过predicate真值检测的元素集合，与filter相反。

every

_.every(list, predicate, [context])
如果list的所有元素都通过了predicate真值检测，就返回true

some

_.some(list, [predicate], [context])
如果list中有通过predicate真值检测的元素，函数中断，返回true，否则返回false

git介绍

git是一种分布式版本控制系统（Distributed Version Control System，简称 DVCS），而SVN是非分布式版本控制系统。

分布式版本控制系统

拥有一个服务端的代码仓库，客户端每次clone都是将远程仓库的代码完整的镜像出来。这样一来，如果服务端的仓库发生故障，事后可以将任何一处镜像出来的本地仓库恢复。
在没有网络的情况下，客户端仍然能够正常的修改，提交代码。

集中式版本控制系统

也拥有一个单一的集中管理的服务器，保存所有文件的修订版本。协同工作的人需要通过客户端连接到服务器，取出最新的代码或者提交更新。
这样做的好处是可以一定程度的看到其他人在做什么。坏处是如果服务器宕机一个小时，这一个小时内任何人都不能够提交代码。

git常用命令行操作

1.如何修改并提交代码
查看代码状态：git status
查看状态简写方式：git status -s
添加所有已修改代码到暂存区：git add .
提交代码并且添加提交信息：git commit -m "first commit"
推送代码到远程仓库：git push origin master

2.git提交时忽略文件
创建一个名为.gitignore的文件，列出要忽略的文件模式。
查看一个 .gitignore 文件的例子：

3.撤销操作
重新提交：git commit --amend
比如第一次提交忘记添加某个文件，需要再次提交：

取消暂存中的某一个文件：git reset HEAD <filename>

撤销对文件的任何修改：git checkout -- <filename>

4.远程仓库操作
查看远程仓库：git remote
你也可以指定选项 -v，会显示需要读写远程仓库使用的 Git 保存的简写与其对应的 URL。

添加远程仓库：git remote add <shortname> <url>

5.打标签
列出已有标签:git tag

查看标签信息：git show <tag>

创建附注标签：git tag -a <v1.0> -m "my version 1.0"

创建轻量标签：git tag <v1.0>
